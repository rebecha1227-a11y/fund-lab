# 亏损体检 + 持仓透视

本文件是 fund-lab 的核心模块。目的：**先诊断病因，再谈调整**。不做涨跌预测，不下买卖指令。

---

## 一、什么时候进入这个模块

用户说类似的话：
- 「我为什么一直亏」「帮我看看问题出在哪」「体检一下我的基金」
- 「我手里这些基是不是买重复了」「我分散吗」
- 「跌了好多，是不是买错了」
- 「要不要加点别的板块对冲」「我缺什么」

---

## 二、真对冲 vs 假对冲（缺口分析的基础）

基金投资者常误以为：**别的板块涨得好 → 买一只那个板块的基 → 对冲**。
这往往是**追高 + 伪分散**，不是对冲。

| 真对冲（低相关，能改善结构） | 假对冲（高相关，加重集中） |
|---------------------------|--------------------------|
| 债基、货币基金 | 已有半导体，再买 CPO/算力/通信 |
| 黄金 | 已有科技链，再买另一个科技子赛道 |
| 宽基（整市场，如沪深300） | 已有纳指，再买第 4 只纳指 |
| 与用户重仓低相关的资产类型 | 名字不同但底层同涨同跌的行业基 |

**缺口分析的核心问题**：不只「你持有什么」，还要「你缺什么、什么重复了、市场变化下该补类型还是该克制」。

---

## 三、体检的底层逻辑：6 条行为损耗

小白亏钱，90% 不是「缺分析工具」，而是中了下面某几条。体检就是**逐条排查**：

| # | 行为损耗 | 典型表现 | 排查问题 |
|---|---------|---------|---------|
| 1 | 追高买入 | 看别人推荐/涨得好才买 | 你是涨了之后才加进去的吗？ |
| 2 | 伪分散 / 集中押注 | 好几只基其实是同一板块 | 穿透后前几大持仓是否高度重叠？ |
| 3 | 频繁申赎 | 跌了赎、涨了追 | 半年内买卖过几次？ |
| 4 | 拿不住 | 一跌就慌、割在低点 | 上次大跌你做了什么？ |
| 5 | 不懂在买什么 | 说不出基金装了什么 | 能说出前十大重仓吗？ |
| 6 | 用错钱 | 短期要用的钱拿去投 | 这笔钱 1-2 年内要用吗？ |

---

## 四、体检流程（5 步）

### 第 1 步：收集持仓（数据铁律：先有数据再分析）

请用户提供，或引导他填 `assets/portfolio-template.md`：
- 每只基金：名称/代码、类型、大致金额或占比、买入时间、当前盈亏
- 这笔钱的用途和时间（1-2 年内是否要用）

信息不全没关系，先用「大致」版本做初步体检，标注哪些结论待精确数据验证。

### 第 2 步：持仓透视（识别伪分散）——本模块最关键

**核心动作：把「基金数量」换算成「真实赌注」。**

1. 给每只基金归类到**底层板块/资产**（不是看基金名字，是看它实际买了什么）。
   - 宽基指数（沪深300、纳指等）→ 看其行业权重，科技权重高的算科技暴露
   - 行业主题基（半导体、白酒、医药、新能源…）→ 直接归入该板块
   - 需要时联网查该基金**最新季报前十大重仓**，标注来源与日期
2. 把高度相关的板块合并成「**赌注群**」。例如：
   - 🔴 AI/半导体链：半导体、半导体设备材料、存储芯片、算力、CPO、通信、科技宽基（纳指等）
   - 🔴 消费链：白酒、食品饮料、消费
   - 🟢 防守/低相关：黄金、债券、红利低波、货币
3. 算出每个「赌注群」的大致仓位占比。
4. 对照「用户以为的分散」和「真实的集中」，画出反差。

**输出这一步时，一定要让用户看到反差**，例如：

```
你以为：分散在 10 个板块
真相：约 X% 押在「AI/半导体」一条链上
```

### 第 3 步：缺口分析（对冲视角）—— `/透视` 必做

在透视完成后，**固定输出这一块**。读 `discipline-rules.md` 里的结构底线对照。

**检查清单：**

| 检查项 | 怎么判断 | 输出什么 |
|--------|---------|---------|
| 单一板块是否超限 | 对照纪律卡（默认 ≤30%） | 超了 → 优先**减集中**，不是加新赛道 |
| 防守资产够不够 | 债+金+现金合计（目标 ≥15%） | 低了 → **补低相关类型**才是真对冲 |
| 有没有宽基 | 是否有整市场指数（沪深300等） | 没有 → 缺口；补宽基比追行业更像分散 |
| 同一赌注重复几只 | 纳指×3、红利×3 等 | 重复 → 合并方向，不是再加同类 |
| 近期市场与用户持仓 | 可结合 `/环境` 或当场联网 | 见下方「环境×缺口」逻辑 |

**环境 × 缺口逻辑（用户问「别的方向好，要不要加」时必用）：**

```
IF 用户重仓板块大跌 AND 其他板块走强：
  1. 先判断：涨好的板块与用户持仓是否低相关？
     - 低相关（如债、金、红利 vs 科技）→ 可能是「补缺口」方向
     - 高相关（科技子赛道互涨）→ 不是对冲，是追高
  2. 再判断：用户是「缺这类资产」还是「已有足够暴露」？
     - 缺 → 可考虑补**类型**（不说具体基金名）
     - 已有 / 超重 → 不建议加，甚至应考虑再平衡
  3. 输出：类型级建议 + 决策问题，不给「买 XX 基」
```

**缺口分析输出示例：**

```
▌你缺什么（对冲视角）
- 重复赌注：纳指×3、红利低波×3、半导体链×4
- 稀缺类型：宽基（整市场）0%；防守（债+金+现金）仅 ~9%，低于 15% 目标
- 真对冲方向：补债/金/宽基（降相关），而非再加科技子赛道
- 若通信/算力近期走强：对你不是对冲，是加重科技暴露 → 建议克制
```

### 第 4 步：逐条排查 6 条行为损耗

对照第二节的表格，结合用户回答和持仓，判断中了哪几条。每条给出：
- 判断（中 / 不中 / 存疑）
- 证据（用户自述 or 持仓数据）
- 为什么这会导致亏损

### 第 5 步：给结论 + 决策要素（不是指令）

- 用一句话总结「亏损的结构性原因」。
- 把用户接下来**该自己想清楚的问题**列出来（提问，不是命令）。例如：
  - 那笔短期要用的钱，要不要先拎出来单独放？
  - 同一板块的多只基，有没有必要合并、少付管理费？
  - 防守资产（黄金/债/红利）占比够不够让你在大跌时拿得住？
- **不要**写「卖 X 买 Y」。

## 六、注意事项

- **温和但直接**：用户亏钱时最需要真话，但要给「还有救」的信心（尤其伤轻时）。
- **归类靠证据**：判断某只基属于哪个板块，优先查真实持仓，别只看名字猜。基金名和实际持仓可能不符。
- **区分账面亏损和真实亏损**：没赎回=账面浮亏，还有回旋余地；被迫赎回=真实亏损。「用错钱」之所以最危险，就是它把浮亏变成实亏。
- **不越界**：体检/透视/缺口分析只负责「看清问题 + 类型级方向」，不负责「开买卖药方」。
- **缺口 ≠ 荐股**：可以说「考虑补债/宽基这类资产」，不能说「买 XX 基金」。
