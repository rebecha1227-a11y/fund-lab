# 环境简报（`/环境`）

命令词：`/环境`、`/市场`。

> **定位**：每周轻量资讯，帮用户**理解**市场，而不是**驱动**交易。
> 只讲和用户**持仓 + 缺口**相关的内容，不做产业链选股，不推荐具体基金。

---

## 一、和「每日选股资讯」的本质区别

| 每日选股/产业链研报 | fund-lab `/环境` |
|-------------------|------------------|
| 扫 20+ 公司、推热门赛道 | 只筛和你持仓相关的 3～5 条 |
| 「买 XX」 | 「对你意味着…（类型级）」 |
| 制造 FOMO、促交易 | 帮理解涨跌、防追高 |
| 服务对象：选股者 | 服务对象：基金组合管理者 |

**第一性**：用户需要的是「懂我为什么涨跌、该不该动」，不是「今天买什么能暴富」。

---

## 二、什么时候进入

- 用户喊 `/环境` 或问「最近市场怎么样」「别的板块涨了要不要加」
- 每周一次（复盘前可自动附带）
- 市场大波动后用户主动问

进入前必读：
- `assets/portfolio-template.md`（持仓与赌注群）
- `assets/discipline-rules.md`（结构底线）
- 可选：`references/loss-checkup.md` 缺口分析逻辑

---

## 三、执行流程（4 步）

### 第 1 步：联网收集（只收相关）

按用户**实际持仓的赌注群**，定向搜索最近 5～7 天资讯。不要泛搜「今日牛股」。

| 用户持仓类型 | 搜索方向（示例） | 目的 |
|-------------|----------------|------|
| 纳指/QDII | 美股科技、美联储、汇率、QDII 额度 | 解释纳指涨跌 |
| A 股科技/半导体 | 半导体板块、存储芯片政策/景气 | 解释科技链波动 |
| 红利低波 | 高股息策略、利率、分红政策 | 解释红利表现 |
| 债/黄金 | 利率、央行、金价 | 解释防守资产 |
| 宏观（通用） | 上证/沪深300 一周表现 | 大环境背景 |

每条资讯必须：**来源 + 日期**。搜不到就说暂无。

### 第 2 步：板块强弱快照（一周）

用表格归纳（数字来自搜索，标注来源）：

| 板块/资产 | 近一周大致表现 | 和用户的关系 |
|----------|---------------|-------------|
| … | … | 你重仓 / 你轻仓 / 你未持有 |

**不做**：预测下周涨跌。

### 第 3 步：挂钩持仓 + 缺口（核心）

必须把市场信息和用户结构连起来，固定回答三个问题：

1. **你最近波动大，主要因为什么？**（哪个赌注群在动）
2. **涨得好的方向，对你算「补缺口」还是「追高」？**（用 loss-checkup 的真对冲/假对冲表）
3. **按你的纪律卡，现在该不该动？**（通常答案：不因一周行情就动结构）

**决策树（必走）：**

```
某板块近期走强
  ├─ 与用户重仓高相关？ → 警告：不是对冲，是加重集中；结合「不追高」纪律
  ├─ 与用户持仓低相关 + 用户缺口里有这类？ → 类型级：可考虑补 XX 类资产（不说具体基金）
  └─ 与用户无关？ → 了解即可，不必行动
```

### 第 4 步：输出简报 + 一句行动建议（非买卖指令）

落盘路径：**`assets/weekly-briefs/YYYY-MM-DD.md`**（文件名与标题日期一致）。  
空白结构见 **`templates/weekly-brief.example.md`**；虚构填写范例见 **`templates/weekly-briefs.example/2026-01-15.md`**。

行动建议只能是：
- 「继续按计划 / 不动」
- 「值得做一次 /透视 看缺口」
- 「想加仓前先走 /纪律」
- 「结构已超限，优先再平衡而非追新板块」

## 四、标准输出模板

```
【环境简报 · {日期} · 近一周】

▌大环境（1～2 句）
[上证/宏观背景，标来源]

▌和你相关的 3～5 条
1. [纳指/QDII：…] 来源：…
2. [科技/半导体：…] 来源：…
3. [红利/债/金：…] 来源：…
（只列你持仓相关的，不列无关热门赛道）

▌板块一周快照
| 板块 | 表现 | 你的暴露 |
|------|------|---------|

▌对你的持仓意味着什么
- 波动来源：[主要哪个赌注群]
- 涨好的方向：[算补缺口 / 算追高 / 无关]
- 对照纪律卡：[板块是否超限、防守是否够]

▌建议（非买卖指令）
[继续定投 / 不动 / 先 /透视 或 /纪律 / 执行既定再平衡计划]

> 以上为学习研究框架，不构成投资建议，最终决策权在你自己。
```

## 五、频率与边界

| 项目 | 规则 |
|------|------|
| 建议频率 | 每周 1 次（用户喊 `/环境` 或复盘前） |
| 不做 | 每日自动推送、产业链选股、推荐具体基金 |
| 不做 | 「必涨/必跌」、制造焦虑 |
| 可做 | 类型级缺口提示（债/宽基/低相关资产） |
| 与 `/透视` 关系 | 环境讲「外面发生什么」；透视讲「里面结构怎样」；常配合使用 |
| 与 `/纪律` 关系 | 用户听完环境想动手 → 转 `/纪律` 刹车 |

## 六、持仓关注清单模板

执行 `/环境` 时，根据 `portfolio-template.md` **动态生成** 3～5 条关注方向，例如：

1. **海外宽基 / QDII** — 汇率、海外市场、申赎状态
2. **A 股科技/成长链** — 板块景气与政策
3. **红利 / 低波策略** — 利率、股息环境
4. **防守（债+金+现金）** — 利率、金价；是否低于自设目标占比
5. **宏观** — 主要宽基指数一周表现

大额调仓或再平衡后，随 portfolio 更新而调整。

## 七、借鉴三个参考仓库的取舍

| 参考 | 借鉴 | 不借鉴 |
|------|------|--------|
| serenity 主题扫描 | 「从市场故事到用户结构」的推理链 | 扫 20+ 公司、输出股票候选 |
| fate 道士宏观 | 先收集数据再下判断；宏观影响板块 | 预测大盘方向、个股择时 |
| daily_stock 情报板 | 结构化输出（结论→数据→行动） | 自动推送、仓位建议 |

fund-lab 的环境简报 = **道士的宏观敏感度 + serenity 的证据标注 + 基金版的缺口视角**，且刻意做「轻量、周频、只相关」。
