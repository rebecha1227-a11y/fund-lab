# 首次使用引导（Onboarding）

> **Agent 必读**：当用户第一次使用 fund-lab，或 `assets/portfolio-template.md` 不存在 / 仍为空白模板时，**必须先执行本流程**，再进入体检、透视等模块。

---

## 一、先跟用户说明的三件事

1. **不连接任何基金账户 API**——所有数据来自用户提供的截图、导出或口述，存在本地 markdown 里。
2. **不预测涨跌、不下买卖指令**——只帮理清结构、纪律和决策要素。
3. **私人数据留在用户自己电脑**——开源包不含你的持仓；`assets/` 和 `QA_Log.md` 请勿提交到公开仓库。

---

## 二、第一次要收集什么（按顺序问）

用表格或清单引导用户补充，**缺什么就问什么**，不要一次甩 20 个问题。

### 1. 交易平台与账户结构

| 要问的 | 示例（仅说明格式，勿照抄为真实持仓） |
|--------|--------------------------------------|
| 在哪些平台买基？ | 如：某银行 App、某第三方理财平台、基金公司直销 App、券商等 |
| 各平台账户是否分开统计？ | 单平台 / 多平台合计看组合 |
| 日常零花钱是否单独放货币基金？ | 若与「投资组合」分开，说明不计入组合占比 |

### 2. 持仓清单（慢变量）

请用户提供**其一**即可（越多越好）：

- 持仓页截图（含基金名称、份额、持有收益）
- 交易记录 / 定投计划截图
- 文字列表：基金全称或代码 + 份额 + 买入方式（定投/一次性）

还要问：

- 每只基金的**购买方式**：日定投 / 周定投 / 月定投 / 一次性 / 已暂停
- 是否有**转换、赎回、待确认**操作
- 是否把某些基金算作同一「赌注群」（如多只同类宽基）

### 3. 收益与总资产口径（快变量）

| 数据 | 通常从哪里来 | 写入哪里 |
|------|-------------|---------|
| 组合总资产 | 各平台「总资产」截图相加（口径一致） | `portfolio-template.md` + `daily-snapshots.md` |
| 当日收益 | 用户常用的「当日盈亏」汇总 App 或平台首页 | `daily-snapshots.md` 的 `daily_profit` |
| 持有 / 累计收益 | 各平台持仓页或资产总览 | 两文件同步 |

**务必确认**：用户「当日收益」以哪个 App/页面为准（只选一个主口径，写入 discipline 备注）。

### 4. 纪律与目标（可选，可第二轮再填）

- 单一板块占比上限、是否接受追高、止盈止损偏好
- 短期是否要用的钱占比
- 写入 `assets/discipline-rules.md`

---

## 三、落盘动作（首次会话结束前应完成）

1. 从 `templates/portfolio-template.example.md` 复制为 `assets/portfolio-template.md`（若尚无）
2. 填写「基本情况」「持仓明细」「赌注群汇总」「定投计划」
3. 初始化 `assets/daily-snapshots.md`（从 `templates/daily-snapshots.example.md` 复制表头）
4. 初始化 `assets/discipline-rules.md`（从 `templates/discipline-rules.example.md`）
5. 可选：`assets/personal-glossary.md`（从 `templates/personal-glossary.example.md`）
6. 创建目录 `assets/weekly-briefs/`、`assets/review-journal/`、`assets/fund-research/`
7. 打开看板：`~/.cursor/skills/fund-lab/dashboard.html`（或工作区 `dashboard.html`）→ **授权本项目文件夹**

---

## 四、推荐用户的第一条命令

完成建档后，建议用户依次尝试：

```
/fund-lab 透视
/fund-lab 环境
/fund-lab 更新今日快照
```

---

## 五、标准开场话术（可改写）

```
欢迎用 Fund Lab。我不会连你的基金账户，需要你用截图或文字告诉我持仓。

先帮我确认三件事：
1）你在哪些平台买基金？（可以多个）
2）发一张持仓总览截图，或列出：基金名 + 份额 + 定投还是一次性
3）你平时看「今天赚/亏多少」用哪个 App 或页面？

拿到后我会写入你本地的 portfolio 档案，并教你看板怎么授权本文件夹刷新。
```
